TDD (Test Driven Development)

O que é TDD?

TDD é uma metodologia de desenvolvimento orientada a testes, ele consiste em escrever testes antes de escrever o código de produção. O objetivo é garantir que o código escrito atenda aos requisitos e que seja de fácil manutenção.

Primeiro faça, depois faça certo, depois faça melhor.

Tipos de testes:

  • Testes de unidade: testam partes específicas do código. Geralmente testam função específicas ou classes.

  • Testes de integração: testam a integração entre diferentes partes do código. Geralmente testam a integração entre função e classes.

  • Testes de aceitação: testam o comportamento do sistema como um todo.

    • Vantagens

    • Testes são escritos antes do código de produção, garantindo que o código atenda aos requisitos.

    • Testes são executados automaticamente, garantindo que o código continue funcionando conforme o esperado.

    • Código de produção é escrito para atender aos testes, garantindo que o código seja de fácil manutenção.

Desvantagens

  • Requer um tempo adicional para escrever os testes.

  • Requer um tempo adicional para escrever o código de produção.

  • Requer um tempo adicional para manter os testes.

Conclusão

TDD é uma abordagem de desenvolvimento de software que consiste em escrever testes antes de escrever o código de produção. O objetivo é garantir que o código escrito atenda aos requisitos e que seja de fácil manutenção.
